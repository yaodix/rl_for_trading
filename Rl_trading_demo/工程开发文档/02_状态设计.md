#### 特征工程

状态计算需要rolling z-score

为什么需要rolling Z-score？

**核心原因：保持特征的平稳性和可比性**

```
# 如果不归一化
特征A（价格）：[380, 390, 400, 410, 420]  # 数值持续增大
特征B（RSI）：[0.3, 0.5, 0.7, 0.5, 0.3]    # 数值在0-1之间

# 模型会误以为特征A更重要（因为数值大）
# 但实际上两个特征同等重要
```

特征工程中rolling z-score涉及参数是滚动窗口大小。

**注意：全局归一化会泄露数据！**


### 状态计算

状态设计也有一个状态空间窗口，它与上述滚动窗口是独立的，一般不相同。

## 📊 两个窗口的本质区别

| 维度                 | 滚动Z-score窗口                  | 状态空间窗口             |
| -------------------- | -------------------------------- | ------------------------ |
| **目的**       | 计算稳定的统计量（均值、标准差） | 为RL模型提供历史上下文   |
| **关注点**     | 统计显著性、稳定性               | 时序模式、趋势识别       |
| **典型大小**   | 较大（22*8≈ 1个月）             | 适中（10-40 ≈ 1-5天）   |
| **更新方式**   | 滚动计算，每个时间点不同         | 固定长度的滑动窗口       |
| **对模型影响** | 影响特征的数值范围               | 影响模型能看到的历史长度 |

状态空间一般是过去一定数量的特征集合，是**智能体在决策时刻能观察到的全部信息**

状态空间参数维度包括：

1. 时间维度：过去多久的信息， bar_count
2. 特征维度：

```
黄金ETF的驱动因素：
1. 价格趋势 → 用收益率序列
2. 多空力量 → 用K线位置、振幅
3. 市场情绪 → 用RSI
4. 风险水平 → 用ATR、波动率
5. 资金动向 → 用成交量比率
```

3. 持仓维度：

自己是什么状态，是否持仓，持有多少。

状态空间必须包含持仓信息！！！

**即使DQN只有买入/卖出两个动作，状态空间也必须包含持仓信息，因为：**

1. **决策依赖当前状态** ：同样的市场信号，满仓和空仓应该采取不同动作
2. **马尔可夫性要求** ：状态必须包含决策所需的所有信息
3. **动作的可执行性** ：需要知道当前仓位才能判断买入/卖出是否合理

已经获取利润率，也可加入。

4. 环境维度：

市场处于什么状态

```
宏观背景的重要性：
- 同样的技术形态，不同环境中意义不同

例子：
- 牛市中突破 → 大概率继续涨
- 熊市中突破 → 可能是假突破
```


### 金融市场状态空间设计

考虑：No st**ationarity of the data（数据的非平稳性）**

**非平稳性**是金融市场数据最本质的特征，也是定义状态空间时**必须特殊考虑**的核心问题。

#### **非平稳性在金融市场的表现**

| 特征               | 含义                                 | 对状态空间的影响                       |
| ------------------ | ------------------------------------ | -------------------------------------- |
| **均值漂移** | 股价长期上涨/下跌                    | 今天的100元和10年前的100元含义完全不同 |
| **方差时变** | 波动率聚集（高波/低波交替）          | 同样的涨跌幅在不同时期代表不同风险水平 |
| **分布变化** | 市场结构突变（政策、危机、制度切换） | 历史分布无法预测未来分布               |

#### **状态空间设计必须应对非平稳性**

如果忽略非平稳性，直接使用**原始价格、原始收益率**作为状态特征，会导致：

#### ❌ **错误做法**

```python
# 直接把价格输入模型
state = [close_price, volume, rsi]
```

**问题**：模型学到的是“价格=100时买入”，但100元在牛市是低价，在熊市是高价，模型无所适从。

#### **正确做法：非平稳性处理技术**

| 方法                  | 原理                 | 示例                          |
| --------------------- | -------------------- | ----------------------------- |
| **滚动归一化**  | 用局部统计量标准化   | (price - 近期均值)/近期标准差 |
| **差分/收益率** | 将价格转为平稳序列   | log(price_t / price_{t-1})    |
| **相对指标**    | 与其他资产的比值     | 股票/大盘、个股/行业          |
| **分位数排名**  | 横截面相对强弱       | 当前价格在历史百分位          |
| **状态识别**    | 显式加入市场状态变量 | 加入VIX、波动率 regime        |

---

## 🎯 **一句话总结**

> **金融状态空间设计的核心不是“如何准确描述当前”，而是“如何在非平稳环境中找到相对稳定的表示”。**
